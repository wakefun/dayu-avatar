# 大宇头像 - 需求方向文档

## 1. 项目背景

「大宇头像」是一个移动端 AI 头像生成应用，UI 设计稿已通过 Stitch 生成。

Stitch 项目地址：

https://stitch.withgoogle.com/projects/18435155359975112615

本项目目标是基于已生成的 UI 设计稿，实现一个完整可运行的 Web 应用。具体实现细节由 Claude Code 在开发阶段进一步拆解和落地，本文件仅描述产品与技术的大方向需求。

---

## 2. 产品定位

「大宇头像」面向希望快速生成高质量个人头像的用户，尤其适合社交媒体创作者、个人品牌用户和需要艺术化头像的普通用户。

用户可以上传个人形象参考图和风格参考图，输入想要的风格描述，系统根据这些信息生成头像，并支持查看结果、保存图库、查看任务进度和历史记录。

---

## 3. UI 与体验方向

整体 UI 风格以已生成的 Stitch 设计稿为准。

核心视觉方向：

- 清透苹果系
- 艺术画廊感
- 清新、高级、简洁
- 富有艺术感
- 液态玻璃质感
- 移动端优先

设计要求：

- 仅需要适配手机端
- 不需要专门兼容 PC 端
- 不使用底部 Tab
- 使用顶部菜单入口和侧边栏导航
- 页面应保持简洁、好用、高级、有审美感

---

## 4. 技术方向

项目采用 monorepo 结构。

推荐技术栈：

- 前端：react
- 后端：nodejs
- 数据库：SQLite
- 图片存储：本地文件目录
- UI 参考：Stitch 生成稿
- ai使用OpenAI 规范接口，通过.env配置baseurl和apikey，图片生成使用`/v1/images/generations`，模型为`gpt-image-2`，默认质量参数为high。

---

## 5. 登录需求

应用需要登录页。

登录方式：

- 使用 OAuth 认证
- oauth端点信息：https://friends.chenyu.cy/oidc/.well-known/openid-configuration
- 登录入口名称为「大宇统一登录」
- 不需要账号密码登录 UI
- 不需要注册表单
- 用户登录后进入头像生成页面

登录页需要体现：

- 应用名称：「大宇头像」
- 登录按钮：「使用大宇统一登录」
- 简洁、高级、可信赖的视觉体验

---

## 6. 核心页面

应用至少包含以下页面：

1. 登录页
2. 头像生成页
3. 生成 Loading / 生成中状态
4. 生成结果展示页
5. 我的图库页
6. 任务队列页
7. 历史记录页
8. 侧边栏菜单
9. 账户设置页

页面布局和视觉细节以 Stitch 设计稿为主要参考。

---

## 7. 头像生成需求

头像生成页是核心页面。

用户需要能够：

- 上传个人形象参考图
- 上传风格参考图
- 输入头像风格描述
- 点击风格标签快速填充或追加提示词
- 设置基础生成参数
- 点击按钮开始生成头像

参考图分为两类：

1. 个人形象参考图  
   用于保留用户个人特征。

2. 风格参考图  
   用于提供头像风格、摄影质感、插画风格或整体氛围。

两类参考图在 UI 上需要明确区分。

---

## 8. 风格标签需求

生成页需要提供可点击的风格标签。

示例标签：

- 清透写真
- 高级杂志
- 艺术肖像
- 法式插画
- 胶片质感
- 水彩插画
- 自然光
- 极简留白
- 温柔奶油色
- 轻奢氛围

交互要求：

- 标签可点击
- 点击后自动填充或追加到提示词输入框
- 已选标签需要有选中态
- 再次点击可取消选择

---

## 9. Loading 与任务状态

用户点击开始生成后，需要有明确的生成中反馈。

Loading 状态需要表达：

- 正在生成头像
- 当前任务仍在处理中
- 可以查看任务队列

可以展示类似步骤：

- 分析个人形象
- 提取风格氛围
- 生成头像构图
- 高清细化中

---

## 10. 任务队列

任务队列页用于展示用户的生成任务。

需要支持的大方向状态：

- 排队中
- 生成中
- 已完成
- 已失败
- 已取消

任务队列中应能看到任务状态、进度或结果入口。

失败任务应支持重试。

---

## 11. 生成结果

生成完成后，需要展示生成头像结果。

用户可以对生成结果进行：

- 查看
- 保存到图库
- 下载
- 重新生成
- 查看生成参数或详情

生成结果页应与整体艺术画廊风格保持一致。

---

## 12. 我的图库

图库页只展示用户主动保存的最终头像作品。

图库主要用于作品管理，不等同于历史记录。

用户应能够：

- 查看已保存头像
- 收藏
- 下载
- 删除
- 查看详情

空状态需要引导用户去生成第一个头像。

---

## 13. 历史记录

历史记录页展示用户所有生成行为，包括成功、失败、取消和重复生成。

历史记录应能体现：

- 生成时间
- 提示词摘要
- 使用的参考图类型
- 任务状态
- 生成参数
- 再次生成入口

历史记录和图库的区别：

- 图库：用户保存下来的最终作品
- 历史记录：用户所有生成任务和生成参数记录

---

## 14. 文件与数据存储

数据库使用 SQLite。

图片文件保存在本地目录，不使用对象存储。

需要保存的数据大方向包括：

- 用户信息
- OAuth 登录信息
- 上传图片记录
- 生成任务记录
- 生成结果记录
- 图库记录
- 历史记录

图片文件大致分为：

- 用户上传的个人形象参考图
- 用户上传的风格参考图
- AI 生成的头像结果
- 可选缩略图

---

## 15. AI 生成能力

AI 头像生成能力可以先抽象为后端生成服务。

开发初期可以先使用 mock 方式跑通完整流程：

```text
上传参考图
→ 创建生成任务
→ 展示 Loading
→ 返回模拟生成结果
→ 保存到图库
→ 查看任务和历史记录
```

后续再接入真实 AI 头像生成服务。

---

## 16. MVP 目标

第一阶段目标是实现一个完整可演示的移动端 Web 应用。

MVP 需要完成：

- react 前端应用
- nodejs 后端服务
- SQLite 数据持久化
- 本地图片上传与访问
- 大宇统一登录入口，可先 mock
- 头像生成主流程，可先 mock 生成结果
- Loading 状态
- 任务队列
- 生成结果展示
- 图库
- 历史记录
- 侧边栏导航
- 基于 Stitch 设计稿还原主要 UI

---

## 17. 开发原则

- 先保证核心流程完整可用
- 具体实现细节由 Claude Code 在开发中拆解
- 优先复刻 Stitch 已生成 UI 的整体视觉与页面结构
- 避免过度设计后台管理、会员系统等非核心功能
- 先用本地 SQLite 和本地文件存储完成 MVP
- 后续再考虑真实 OAuth、真实 AI 服务、图片审核、订阅系统等扩展能力
